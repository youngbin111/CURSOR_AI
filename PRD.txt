### **1. 문서 개요**

**제품명** : CleanBoost (가칭)

**목표**

- 이 문서는 CleanBoost 웹 솔루션의 개발 목표, 핵심 기능, 사용자 요구사항, 그리고 성공 기준을 명확히 정의하여 프로젝트 팀이 공통된 비전을 가지고 개발에 임할 수 있도록 합니다.

---

### **2. 비전 및 목표**

**비전**

: 컴퓨터 사용이 어렵고 복잡하게 느껴지는 일반 사용자도 자신의 노트북을 전문가처럼 관리하고 최적의 성능을 유지할 수 있게 돕는 가장 쉽고 직관적인 도구.

**문제점**

: 많은 사용자들이 노트북 성능 저하의 원인을 파악하지 못하고, 불필요한 파일을 수동으로 찾아 삭제하는 데 어려움을 겪습니다. 이로 인해 소중한 시간과 하드디스크 공간을 낭비하고 있습니다.

**해결책**:

- **시각적 진단**: 사용자가 자신의 시스템 상태(CPU, RAM)를 한눈에 파악할 수 있는 대시보드를 제공합니다.
- **불필요 파일 안내**: 노트북에 쌓인 불필요한 파일을 자동으로 식별하고, 사용자에게 **삭제를 권장하는 파일 목록과 삭제 방법**을 안내합니다.
**전략**: 웹 기반으로 접근성을 높이고, 직관적인 UI/UX를 통해 기술 지식이 없는 사용자도 쉽게 사용할 수 있도록 설계합니다.

---

### **3. 사용자 스토리 (User Stories)**

사용자: 노트북을 쾌적하게 사용하고 싶은 대학생

- **시스템 모니터링**:
    - "내 노트북이 갑자기 느려졌을 때, **어떤 프로그램이 CPU를 많이 잡아먹는지 알고 싶어요.** 그래야 그 프로그램을 끌 수 있으니까요."
    - "평소에 컴퓨터를 켜놓고 다닐 때, **메모리가 얼마나 사용되고 있는지** 한눈에 보고 싶어요. 내가 너무 많은 창을 띄워 놓는 건 아닌지 궁금해요."
- **찌꺼기 파일 클리너**:
    - "디스크 용량이 부족하다고 알림이 떴을 때, **어떤 파일들을 지워야 할지 알려줬으면 좋겠어요.** 내가 직접 찾아서 지우기에는 너무 번거로워요."
    - "웹사이트 방문 기록이나 임시 파일 같은 불필요한 찌꺼기들을 알려주면, **내가 직접 찾아서 지우고 싶어요.**"

### **3-1. 타겟 사용자**

**메인 사용자 페르소나**:

- **이름**: 사용자
- **기술 지식**: 낮음. 일상적인 용도로 노트북을 사용하며, 컴퓨터 성능 저하의 원인 파악과 해결에 어려움을 느낀다.
- **문제점**: 과제 파일이 많아지며 용량이 부족해지고 속도가 느려지지만, 어떤 파일을 지워야 할지 모른다.

---

### **4. 기술 스택 선호도**

- **언어 및 프레임워크**: Python을 선호하며, API 서버 구축에는 **FastAPI** 또는 **Flask**를 사용합니다.
- **실시간 통신**: 노트북의 상태 정보를 실시간으로 주고받기 위해 **python-socketio** 또는 **Flask-SocketIO** 라이브러리를 활용합니다.
- **로컬 에이전트**: 사용자의 시스템에 설치될 보조 프로그램은 **Python**으로 개발되며, 독립적인 패키지 형태로 배포됩니다.
- **데이터 시각화**: `Chart.js`를 사용합니다.

### **5. 기능 요구사항 (Functional Requirements)**

- **FR-1: 실시간 시스템 모니터링 대시보드**
    - **FR-1.1**: 사용자 웹 브라우저 접속 시, CPU, RAM 사용량을 **실시간으로 그래프 또는 숫자**로 표시해야 합니다.
    - **FR-1.2**: 각 그래프에 현재 사용량(%)과 실제 수치(예: 8GB / 16GB)를 함께 표기해야 합니다.
    - **FR-1.3**: 노트북 스토리지(SSD/HDD)의 **총 용량, 사용 중인 용량, 남은 용량**을 원형 그래프와 수치로 보여줘야 합니다.
- **FR-2: 찌꺼기 파일 스캔 및 삭제**
    - **FR-2.1**: **'스캔 시작' 버튼**을 제공하여 사용자가 수동으로 스캔을 시작할 수 있게 합니다.
    - **FR-2.2**: 스캔 완료 후, **'브라우저 캐시', '시스템 임시 파일', '휴지통'** 등 항목별로 그룹화하여 목록을 보여줘야 합니다.
    - **FR-2.3**: 각 항목 옆에 **확보 가능한 용량**과 함께 **체크박스**를 제공하여 사용자가 삭제할 파일을 직접 선택하게 합니다.
    - **FR-2.4**: **'선택 항목 삭제' 버튼**을 제공하고, 클릭 시 선택된 파일들을 안전하게 삭제해야 합니다.
    - **FR-2.5**: 파일 삭제 완료 후 '총 확보된 용량: XX GB'와 같은 성공 메시지를 사용자에게 보여줘야 합니다.

### **6. 비기능 요구사항 (Non-Functional Requirements)**

- **성능**:
    - NFR-1.1: 웹페이지 로딩 시간은 **5초 이내**여야 합니다.
    - NFR-1.2: 
    시스템 모니터링 데이터 업데이트 간격은 **1초 미만**이어야 합니다.
    - NFR-1.3: 찌꺼기 파일 스캔 시간은 일반적인 노트북 환경에서 **30초 이내**여야 합니다.
- **보안**:
    - NFR-2.1: 사용자 로컬 시스템에 직접 접근하는 기능을 구현하기 위해 안전한 통신 프로토콜을 사용해야 합니다.
    - NFR-2.2: 사용자의 개인 파일(문서, 사진 등)은 절대 스캔하거나 삭제해서는 안 됩니다. 오직 임시 파일과 캐시 등 찌꺼기 파일만 다뤄야 합니다.
- **사용성**:
    - ~~NFR-3.1: 모바일 환경에서도 접근 가능한 반응형 웹 디자인을 적용해야 합니다.~~
    - NFR-3.2: 기술 용어를 최소화하고 직관적인 아이콘과 라벨을 사용해야 합니다.

### **7. 성공 지표 (Success Metrics)**

- **월간 활성 사용자(MAU)**: 월 10,000명 이상
- **클리너 기능 사용률**: 월간 활성 사용자 중 30% 이상이 클리너 기능을 사용
- **페이지 체류 시간**: 평균 2분 이상
- **재방문율**: 1주일 내 재방문율 10% 이상

이 PRD는 웹에서 직접 삭제하는 대신 '안내 및 가이드'를 제공하는 방식으로 변경하여, 기술적 구현 난이도를 낮추고 사용자에게 더 안전하고 명확한 경험을 제공합니다. 💻
